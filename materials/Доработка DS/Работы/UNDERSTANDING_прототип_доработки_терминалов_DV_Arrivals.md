## 1. Что требуется сделать

### 1.1. Суть доработки

Плагин **DigitalVoice** (озвучка событий Front) должен получить возможность выводить звук на экраны **Arrivals** (электронная очередь). Для этого администратору нужен обновлённый интерфейс в iikoWeb.

**Задача прототипа:** модифицировать существующий экран «Настройка терминалов» в прототипе `web-screens`, чтобы показать, как будет выглядеть интерфейс после доработки.

### 1.2. Бизнес-контекст (из стенограммы встречи)

На встрече 26.06.2026 Руслан, Михаил, Ваня, Даша и Кирилл обсудили:

- **Проблема:** звуковые оповещения с кассы не слышны на кухне/выдаче, где персонал работает с экранами Arrivals
- **Решение:** плагин DigitalVoice должен отправлять команды воспроизведения звука на плагин Arrivals через механизм ExternalOperations SDK V9
- **Ключевое решение по UI (Михаил, 12:29):** в разделе «Терминалы» аудиоустройство Arrivals отображается как устройство вывода. При его выборе раскрывается дерево дисплеев. Каждому дисплею назначаются обработчики событий
- **Руслан (13:04):** «Можно и так. Да, можно и так»
- **Принято единогласно:** Arrivals как аудиоустройство + дерево дисплеев + привязка обработчиков к дисплеям

### 1.3. Что показать в прототипе (UI-изменения)

На странице «Настройка терминалов» (текущий файл: `sounds-terminals-screen.component.ts`) нужно:

1. **Расширить выпадающий список «Аудиоустройство»** — добавить пункты «Arrivals {login}» с иконкой монитора, отделённые разделителем от физических устройств
2. **При выборе Arrivals — показать дерево дисплеев** (уровень 3) под строкой терминала:
   - Имя дисплея + индикатор «Онлайн» (зелёный) / «Офлайн» (серый)
   - Мультиселект обработчиков для каждого дисплея
3. **Кнопка «Обновить»** — для ручного обновления списка дисплеев
4. **Состояния:** загрузка дисплеев, ошибка загрузки, офлайн-дисплей
5. **Сохранение** — кнопка «Сохранить» учитывает привязки обработчиков к дисплеям
6. **Поиск** — учитывает имена дисплеев

## 2. Где и как реализовать

### 2.1. Целевой прототип

**Прототип:** `web-screens` (slug: `web-screens`, уже существует)
**Экран для доработки:** `src/app/prototypes/web-screens/screens/sounds-terminals-screen.component.ts`

### 2.2. Текущее состояние экрана (AS-IS)

| Элемент | Текущая реализация |
|---------|-------------------|
| Структура | Группы ресторанов → раскрываются → таблица терминалов (3 колонки) |
| Колонки | «Терминал», «Обработчики» (multiselect dropdown), «Аудиоустройство» (select) |
| Аудиоустройства | Список `AUDIO_DEVICES`: «Динамики (High Definition Audio Device)», «Device 1», «Device 2» |
| Данные | `SoundTerminal { id, name, lastActivity, handlerIds, audioDevice }` |
| Сохранение | Через `StorageService` в localStorage |

### 2.3. Что изменить (TO-BE)

#### 2.3.1. Типы данных (types.ts)

Добавить новые интерфейсы:

```typescript
// Новые типы для Arrivals-дисплеев
export interface ArrivalsDevice {
  moduleId: number;
  login: string;          // Имя кассы (напр. "Касса-1")
  isOnline: boolean;
  displays: ArrivalsDisplay[];
}

export interface ArrivalsDisplay {
  id: string;             // UUID дисплея
  name: string;           // Человекочитаемое имя
  isOnline: boolean;
  handlerIds: number[];   // Назначенные обработчики
  audioOutput: 'tv' | 'centralized';  // Источник звука
}

// Расширение SoundTerminal
export interface SoundTerminal {
  // ... существующие поля
  arrivalsDeviceId?: number;  // moduleId выбранного Arrivals (или null)
}
```

#### 2.3.2. Мок-данные (mock-data.ts)

Добавить:
- `MOCK_ARRIVALS_DEVICES` — 2-3 экземпляра Arrivals с дисплеями
- Обновить `AUDIO_DEVICES` — добавить пункты «Arrivals {login}»
- Обновить `MOCK_SOUND_TERMINAL_GROUPS` — для одного терминала выбрать Arrivals как аудиоустройство

#### 2.3.3. Экран (sounds-terminals-screen.component.ts)

Основные изменения в шаблоне и логике:

**Шаблон:**
1. В `<select>` аудиоустройства:
   - Группировка: физические устройства → разделитель → «Arrivals {login}»
   - Для пунктов Arrivals — иконка монитора вместо динамика
   - Серый цвет для офлайн-Arrivals с меткой «(офлайн)»

2. После строки терминала (`<tr>`) — при выбранном Arrivals:
   - Строки дисплеев (`<tr class="display-row">`) с отступом 24px
   - Левая граница-индикатор (вертикальная линия)
   - В каждой строке: имя дисплея + статус (зелёная/серая точка) + мультиселект обработчиков

3. Кнопка «Обновить» — рядом с выбранным «Arrivals {login}»

4. Состояния:
   - Загрузка: `<tr>` с текстом «Загрузка списка дисплеев...» + спиннер
   - Ошибка: `<tr>` с текстом ошибки + кнопка «Обновить»

**Логика (TypeScript):**
- Массив `arrivalsDevices: ArrivalsDevice[]`
- Метод `selectArrivalsDevice(terminalId, moduleId)` — выбор Arrivals для терминала
- Метод `refreshDisplays(moduleId)` — обновление списка дисплеев
- Метод `toggleDisplayHandler(displayId, handlerId)` — назначение обработчика на дисплей
- Фильтрация поиска с учётом имён дисплеев

## 3. Ключевые решения из спецификации

### 3.1. Что НЕ входит в scope прототипа

| Исключено | Причина |
|-----------|---------|
| MenuBoard как устройство вывода | Вне scope спецификации |
| Реальная интеграция с ExternalOperations | Это прототип — всё на мок-данных |
| Backend-логика (API, БД) | Прототип — только фронт |
| Тема «Конфигуратор плагинов» | Отдельная задача (оптимизация) |

### 3.2. Дизайн-решения для UI

| Решение | Источник |
|---------|----------|
| Arrivals как пункт в выпадающем списке аудиоустройств | Михаил (12:29), Руслан (13:04) |
| Дерево дисплеев под строкой терминала | Михаил (12:41-13:32) |
| Мультиселект обработчиков для каждого дисплея | Спецификация, раздел 7.2.3 |
| Разделитель между физическими и виртуальными устройствами | Спецификация, раздел 8.4.3 |
| Отступ 24px для строк дисплеев | Спецификация, раздел 8.5 |
| Индикаторы «Онлайн»/«Офлайн» с цветовыми точками | Спецификация, раздел 8.5 |
| Кнопка «Обновить» для ручного обновления дисплеев | Спецификация, раздел 7.2.2 #9 |

### 3.3. Состояния UI (диаграмма из спецификации, раздел 8.11)

```
S0 (нет Arrivals) → S1 (Arrivals обнаружены) → S2 (загрузка дисплеев)
S2 → S3 (дисплеи загружены) → S5 (несохранённые изменения) → S6 (сохранено)
S2 → S4 (ошибка загрузки) → S2 (кнопка «Обновить»)
S3 → S8 (Arrivals офлайн) → S3 (восстановление)
```
