## Ответ: как правильно

### Меню-лист (MenuList) — ЭЛЕМЕНТ ТЕМЫ

| Аспект | Правильная реализация |
|--------|----------------------|
| **Тип** | Элемент темы (`type: "MenuList"`) |
| **Где размещается** | Непосредственно в `screens[].elements[]` — на canvas, как Text/Image/Advertise |
| **Как добавляется** | «Добавить элемент» → «Меню-лист» в палитре конструктора |
| **Что делает** | Таблица строк: иконка + название + модификаторы/размеры + цена |
| **Является ли контролом** | НЕТ. Это самостоятельный элемент, не требует ControlArea |

### Контрол меню-борда — КОНТРОЛ ВНУТРИ CONTROLAREA

| Аспект | Правильная реализация |
|--------|----------------------|
| **Тип** | Контрол внутри ControlArea |
| **Где размещается** | Внутри area-элемента (ControlArea) |
| **Как добавляется** | Добавить «Область» → в инспекторе выбрать контрол «Меню-борд» из выпадающего списка |
| **Что делает** | Карточки блюд: изображение + название + цена |
| **Является ли элементом темы** | НЕТ. Не может существовать вне ControlArea |

---

## Что реализовано в прототипе (соответствие)

| Компонент | Реализация | Статус |
|-----------|-----------|:-----:|
| **Меню-лист** | Элемент `type: 'menulist'` в `ArrivalsElementType`, палитра «Добавить элемент» → «Меню-лист», canvas + инспектор свойств | ✅ Верно |
| **Контрол меню-борда** | Мок-контрол id=307 в `MOCK_ARRIVALS_CONTROLS`, выбирается через выпадающий список «Контрол» в AreaElementInspector | ✅ Верно |

---

## Вывод

Архитектура реализована **правильно**:

- **MenuList** = элемент темы (как Text, Image, Advertise, Price)
- **Контрол меню-борда** = контрол внутри ControlArea (как «Состав заказа»)

Это соответствует и спецификации, и итогам встречи 08.06.2026. Ошибок в архитектуре нет.
