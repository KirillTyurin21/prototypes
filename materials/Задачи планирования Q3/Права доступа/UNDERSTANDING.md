# Понимание задачи: Прототип «Права доступа DS»

Дата: 2026-06-10

---

## 1. Что нужно сделать (кратко)

Создать интерактивный прототип страницы **«Права доступа»** из Web, с доработками по спецификации **DS-Права-доступа**, и разместить его в отдельном разделе на главной странице прототипов.

---

## 2. Анализ спецификации

### 2.1. Суть задачи (из спецификации)

Сейчас в Web зарегистрированы только два кода DS: `APP_ARRIVALS` и `APP_DIGITAL_VOICE`. Оба висят на корневом уровне дерева прав, без папки. Коды `APP_CUSTOMER_SCREEN`, `APP_HINTS`, `APP_MENU_BOARD` не зарегистрированы. Бэкенд не проверяет права DS.

**Цель:** сгруппировать права DS в единую папку «Экраны и звуки», зарегистрировать недостающие коды, добавить бэкенд-проверку.

### 2.2. Целевая структура (из спецификации)

```
Экраны и звуки/                     ← НОВАЯ папка верхнего уровня
  Экран покупателя/
    APP_CUSTOMER_SCREEN             ← зарегистрировать (новый код)
  iikoArrivals/
    APP_ARRIVALS                    ← перенести с корневого уровня
  Подсказки/
    APP_HINTS                       ← зарегистрировать (новый код)
  Озвучка/
    APP_DIGITAL_VOICE               ← перенести с корневого уровня
  MenuBoard/
    APP_MENU_BOARD                  ← зарегистрировать (новый код)
```

### 2.3. Что делает прототип

Прототип **визуализирует целевую структуру** прав доступа — то, как страница «Права доступа» будет выглядеть ПОСЛЕ внедрения изменений:

- Новая папка «Экраны и звуки» верхнего уровня (в ряду с «Гости», «Бизнес» и др.)
- 5 подпапок внутри
- 5 кодов `APP_*` размещены в соответствующих подпапках
- Три новых кода (`APP_CUSTOMER_SCREEN`, `APP_HINTS`, `APP_MENU_BOARD`) — с пометкой «Новый»

Прототип **НЕ реализует** бэкенд-логику, это витрина для демонстрации UI.

## 4. Что нужно создать

### 4.1. Модификация главной страницы (`home.component.ts`)

На главной странице `http://localhost:3000/` нужно:

1. **Создать блок «WEB»** — отдельная секция с заголовком, группирующая web-прототипы:
   - **Advertise Screens** → ведёт на `/prototype/web-screens/displays` (уже существует)
   - **Общие настройки** → ведёт на новый прототип `/prototype/web-settings`

2. Блок «Front» — для существующих Front-прототипов (опционально, или оставить как есть)

**Вариант реализации:** добавить категории в `PrototypeEntry` (поле `category: 'front' | 'web'`), на главной показывать сгруппированно.

### 4.2. Новый прототип `web-settings`

**Slug:** `web-settings`  
**Кодовое имя:** не требуется (внутренний прототип, не клиентский)  
**Путь:** `http://localhost:3000/prototype/web-settings`

#### Структура

```
src/app/prototypes/web-settings/
├── web-settings.routes.ts               ← Маршруты
├── web-settings-prototype.component.ts  ← Корневой компонент (sidebar + router-outlet)
├── changelog.data.ts                    ← История изменений
├── components/
│   └── web-settings-sidebar.component.ts ← Sidebar в стиле Web (Общие настройки)
├── screens/
│   └── permissions-screen.component.ts  ← Экран «Права доступа»
├── data/
│   └── mock-permissions.ts              ← Мок-данные: роли, дерево прав
└── types.ts                             ← Типы (PermissionNode, Role, etc.)
```

#### Экран «Права доступа»

Повторяет UI стенда (sidebar в стиле Web + контент permissions), НО с целевой структурой прав:

- **Левая панель:** список ролей (radio), поиск
- **Правая панель:** дерево прав с папками и чекбоксами
  - Папки верхнего уровня (сворачиваемые, folder icon, название, checkbox)
  - Включая НОВУЮ папку «Экраны и звуки» с 5 подпапками
  - 3 новых кода помечены бейджем «Новый»
- **Кнопка «Сохранить»** — сохраняет состояние в localStorage
- **Кнопка «Массовое редактирование прав»** — не функциональна (заглушка)
- **Поиск** по названию/коду — фильтрует дерево

### 4.3. Регистрация (3 файла)

1. `prototypes.registry.ts` — добавить запись `web-settings`
2. `app.routes.ts` — добавить `loadChildren`
3. `icons.module.ts` — проверить нужные иконки (shield, folder, key — скорее всего уже есть)

## 7. Моё предложение

- Разместить «Общие настройки» как отдельный прототип `web-settings` внутри Web-блока (не дочерний экран web-screens)
- На главной — две кнопки в блоке «WEB»: «Advertise Screens» и «Общие настройки»
- При клике на «Общие настройки» → открывается страница с Web-дизайном, левый sidebar «Настройки и обслуживание системы → Общие настройки», контент = «Права доступа» с целевым деревом
- Кнопка «Сохранить» — сохраняет состояние чекбоксов в localStorage (работает)
