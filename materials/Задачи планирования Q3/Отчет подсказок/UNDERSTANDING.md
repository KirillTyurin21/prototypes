# Понимание задачи: Прототип «Отчет подсказок»

Дата: 2026-06-10

---

## 1. Что нужно сделать (кратко)

Доработать существующий прототип **web-screens/hints** — добавить в карточку подсказки (drawer) кнопку **«Отчет»**, которая открывает **полноэкранную панель** с дашбордом эффективности подсказки. Все по спецификации **DS-Отчет-подсказок**.

---

## 2. Анализ спецификации

### 2.1. Суть задачи (из спецификации)

Подсказки (Hints) — механизм кросс-продаж на Customer Screen. Сейчас администратор **не имеет инструмента** для оценки эффективности подсказок. Спецификация добавляет:

| # | Функция | Суть |
|---|---------|------|
| 1 | Запись факта показа в заказ | Плагин пишет в `OrderExternalData` ключ `hint_{id}` (это НЕ прототип — сторона плагина) |
| 2 | **Дашборд в карточке подсказки** | Кнопка «Отчет» → полноэкранная панель с KPI: Показы, Срабатывания, Конверсия |
| 3 | Фильтрация по периоду | Неделя (по умолчанию) / Месяц. Без перезагрузки страницы |

### 2.2. Целевой UI (из спецификации, раздел 7)

**Кнопка «Отчет»:**
- Размещается в drawer карточки подсказки, в верхней части (рядом с кнопкой «Сохранить»)
- Доступна **только для уже сохраненных** подсказок (НЕ при создании новой)

**Полноэкранная панель дашборда:**
- Открывается анимацией slide-out справа
- Перекрывает **всю рабочую область** (таблицу подсказок), но НЕ левое меню и НЕ header
- Структура:
  - **Заголовок**: название подсказки + кнопка «Назад» (стрелка влево)
  - **Фильтр периода**: выпадающий список «Неделя» / «Месяц»
  - **KPI-карточки** (3 шт. в ряд): Показов, Срабатываний, Конверсия
- По умолчанию загружается период «Неделя»
- Закрывается: кнопка «Назад», клик вне панели, Escape

**Состояния:**
| Состояние | Поведение |
|-----------|-----------|
| Нет данных | Нули + текст «Нет данных за выбранный период» |
| Ошибка загрузки | «Не удалось загрузить статистику. Попробуйте позже.» + кнопка «Повторить» |
| Подсказка не найдена | «Подсказка не найдена» |

### 2.3. API-контракт (из спецификации, раздел 8.3)

```
GET /api/hints/{hintId}/report?period={period}
```
Ответ: `{ shows: 150, conversions: 45, conversionPercent: 30.0 }`

В прототипе — **мок-данные**, которые меняются при смене периода (неделя/месяц).

### 2.4. Что НЕ входит в scope прототипа

- Глубокая аналитика (год, сравнение периодов) — это в OLAP-отчеты
- Сравнение нескольких подсказок
- Экспорт в Excel/CSV
- Детализация по ресторанам
- Реальная интеграция с backend/OLAP — прототип работает на мок-данных

## 3. Анализ фактической страницы (стенд)

### 3.1. Страница `stand2-iikoweb/customer-screen/index.html#/recommendation`

Страница имеет стандартную структуру Web:
- **Левое меню** (sidebar): «Экраны и звуки» → «Подсказки» (активный пункт)
- **Верхняя панель** (header): уведомления, поиск, профиль «Administrator»
- **Заголовок**: «Подсказки» + кнопки «Создать папку Pro», «Добавить»
- **Таблица** подсказок: колонки Название, Период действия, Время действия, Статус, Действия
- В таблице — папки и подсказки. Есть папки: «папка с подсказками», «Вечерние подсказки», «папка внутри 6», «test45», «Процент и фикс сумма»
- Действия: редактировать (карандаш), удалить (корзина), переместить (стрелки)

**Редактор подсказки** (drawer) — выезжает справа, содержит:
- Основные параметры (название, даты, время)
- Связка блюд (триггеры → рекомендация)
- Слоган рекомендации
- Скидка (тип, размер, привязка к скидке из справочника)
- Контрол подсказки
- Footer: Отмена + Сохранить

**Кнопки «Отчет» на фактической странице НЕТ** — это и есть доработка по спецификации.

## 5. План реализации (по шагам)

### Шаг 1: Мок-данные для отчета

Создать функцию, возвращающую `{ shows, conversions, conversionPercent }` в зависимости от `hintId` и `period`. Данные — реалистичные:
- Неделя: ~100–200 показов, 20–40% конверсия
- Месяц: ~400–800 показов, конверсия примерно та же
- Для разных подсказок — разные цифры

### Шаг 2: Компонент `HintReportPanelComponent`

Новый standalone-компонент в `components/hint-report-panel/`.

**Входные параметры:**
- `@Input() open: boolean`
- `@Input() hint: Hint | null`

**Выходные события:**
- `@Output() close = new EventEmitter<void>()`

**Структура:**
```
┌─ Overlay (полупрозрачный фон) ─────────────────────────────────┐
│  ┌─ Панель (slide-out справа, шире drawer — max-w-4xl) ────┐  │
│  │  Header: ← Назад    |    «Название подсказки»            │  │
│  │  ─────────────────────────────────────────────────────  │  │
│  │  Период: [Неделя ▾]                                     │  │
│  │                                                          │  │
│  │  ┌──────────┐  ┌──────────────┐  ┌──────────────┐      │  │
│  │  │ Показов  │  │ Срабатываний │  │  Конверсия   │      │  │
│  │  │   150    │  │     45       │  │    30%       │      │  │
│  │  └──────────┘  └──────────────┘  └──────────────┘      │  │
│  └────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

**Состояния:**
- `loading` — скелетоны/спиннер при «загрузке» данных
- `empty` — нули + «Нет данных за выбранный период»
- `error` — «Не удалось загрузить статистику» + кнопка «Повторить»
- `normal` — KPI-карточки с данными

**Имитация загрузки:** при открытии и смене периода — искусственная задержка 600–800 мс (setTimeout), чтобы показать loading state.

### Шаг 3: Кнопка «Отчет» в drawer

Добавить секцию `unreleased` в `web-screens/changelog.data.ts`.

## 7. Ожидаемый результат

После реализации:
1. Открываем прототип → Web Screens → Подсказки
2. Кликаем на строку подсказки → открывается drawer редактирования
3. В footer drawer видна кнопка «Отчет» (только для существующих подсказок)
4. Нажимаем «Отчет» → открывается полноэкранная панель с дашбордом
5. Панель показывает KPI: Показов, Срабатываний, Конверсия за неделю
6. Переключаем период на «Месяц» → цифры обновляются (с анимацией загрузки)
7. Закрываем панель (кнопка «Назад», Escape, клик вне) → возвращаемся к drawer
