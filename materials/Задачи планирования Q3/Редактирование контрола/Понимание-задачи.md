## 1. Суть задачи

### Как работает сейчас

- Редактирование контрола происходит через **отдельный справочник контролов** (вне темы): 
  Администратор → Справочник контролов → Находит контрол → Редактирует → Изменения применяются везде.
- В теме контрол можно только **прикрепить** к области (area), но нельзя редактировать его свойства прямо из редактора темы.
- В редакторе темы при клике на область открывается **инспектор области** (area-element-inspector) — там можно выбрать контрол из выпадающего списка, настроить режим области, статусы, сортировку. Но сами элементы контрола (текст, картинка, цена и т.д.) редактировать **нельзя**.

### Что нужно сделать

Добавить возможность **редактировать контрол прямо из редактора темы** — в инспекторе области (area-element-inspector) появляется кнопка «Редактировать контрол», которая открывает существующий редактор контрола (`arrivals-control-editor`). После редактирования администратор возвращается обратно в редактор темы.

### Два режима сохранения (из спецификации)

| Режим | Поведение | Use case |
|-------|-----------|----------|
| **Сохранить везде** | Изменения применяются к самому контролу. Все темы с этим контролом получают обновления | «Поменять контрол глобально» |
| **Сохранить как копию** | Создаётся новый контрол на основе edited, исходный не меняется. Область в теме перепривязывается к новому контролу | «Хочу поэкспериментировать, не ломая оригинал» |

## 2. Что будем делать (границы прототипа)

### Объём: только Тема Электронной очереди

Для примера используем **только** редактирование контрола из **Темы Электронной очереди** (Arrivals Theme). Это экран `arrivals-theme-editor-screen.component.ts`, маршрут `arrivals-theme-editor/:id`.

### Что именно реализуем

1. **Кнопка «Редактировать контрол»** в инспекторе области (area-element-inspector) — появляется если у области выбран контрол (`areaControlId`). По нажатию → переход на экран `arrivals-control-editor/:id`
2. **Редактор контрола** — существующий экран `arrivals-control-editor-screen.component.ts`. Уже умеет: canvas с элементами, drag-and-drop, инспектор свойств, сохранение
3. **Возврат обратно** — после сохранения в редакторе контрола, кнопка «Назад» возвращает в редактор темы

### Что НЕ делаем

- Не трогаем редактор тем для Доски меню (menuboard-theme-editor) — только Электронная очередь
- Не создаём новое модальное окно — используем существующий редактор контрола
- Не делаем «Контрол в контроле» (вложенность) — только первый уровень
- Не трогаем серверную часть / API — всё на мок-данных и localStorage

### Детали UX

- Кнопка «Редактировать контрол» находится в `area-element-inspector` (правая панель), в секции выбора контрола
- Кнопка видна **только** когда у области выбран контрол (`areaControlId` не пустой)
- По нажатию — роутинг на `/prototype/web-screens/arrivals-control-editor/:controlId`
- Контрол открывается в том же макете (sidebar + top-bar), что и редактор темы
- После редактирования пользователь жмёт «Назад» (или «Сохранить») и возвращается в редактор темы

## 3. Архитектура: что и где меняем

### Существующие файлы, которые будем менять

| Файл | Что меняем |
|------|-----------|
| `components/theme-editor/area-element-inspector.component.ts` | Добавляем кнопку «Редактировать контрол» в секцию выбора контрола. Кнопка видна только при выбранном `areaControlId`. По клику — `@Output() editControl = new EventEmitter<number>()` |
| `screens/arrivals-theme-editor-screen.component.ts` | Подписываемся на `editControl` от area-element-inspector. По событию — роутинг на `arrivals-control-editor/:controlId` |

### Новые файлы

**НЕ создаём.** Используем существующий `arrivals-control-editor-screen.component.ts`.

### Как встроим в существующий flow

```
canvas area element (click)
  → открывается area-element-inspector (уже работает)
  → в инспекторе видна кнопка «Редактировать контрол» (если areaControlId задан)
  → по клику → router.navigate(['/prototype/web-screens/arrivals-control-editor', controlId])
  → открывается существующий редактор контрола (canvas + элементы + инспектор)
  → пользователь редактирует → «Сохранить» → «Назад» → возврат в редактор темы
```

## 4. План реализации (шаги)

### Шаг 1. Доработать `area-element-inspector.component.ts`
- Добавить `@Output() editControl = new EventEmitter<number>()`
- В секции выбора контрола (под `<select>` с `areaControlId`) добавить кнопку:
  ```html
  <button *ngIf="element.areaControlId" class="btn-edit-control" (click)="editControl.emit(element.areaControlId)">
    <lucide-icon name="pencil" [size]="14"></lucide-icon>
    Редактировать контрол
  </button>
  ```
- Кнопка стилизована как ссылка/outlined, не перегружает интерфейс

### Шаг 2. Подписаться на событие в `arrivals-theme-editor-screen.component.ts`
- В шаблоне: `(editControl)="onEditControl($event)"` на `<app-area-element-inspector>`
- Метод `onEditControl(controlId: number)`: `this.router.navigate(['/prototype/web-screens/arrivals-control-editor', controlId])`

### Шаг 3. Проверить возврат из редактора контрола
- Убедиться, что в `arrivals-control-editor-screen` кнопка «Назад» работает корректно
- При возврате тема должна перезагрузиться с актуальными данными контрола (из localStorage)

### Шаг 4. Интеграция и проверка
- Проверить полный цикл: редактор темы → кнопка → редактор контрола → сохранить → назад → редактор темы
- Проверить, что изменения контрола видны на canvas после возврата
- Проверить работу Reset (сброс данных)
- Обновить `changelog.data.ts`

## 5. Данные и типы

### Текущая модель ArrivalsControl
```typescript
interface ArrivalsControl {
  id: number;
  name: string;
  statusType: 'kitchen' | 'delivery' | 'balancer';
  elements: ArrivalsThemeElement[];  // ← это то, что будем редактировать
}
```

### Текущая модель ArrivalsThemeElement (элемент контрола)
Включает поля: id, type, name, x, y, width, height, borderWidth, borderColor, borderRadius,
text, fontFamily, fontSize, fontBold, fontItalic, textAlign, imageUrl, 
и множество других опциональных полей (см. `types.ts`).

### Существующие мок-данные
- `MOCK_ARRIVALS_CONTROLS` — 6 контролов (id 302-307)
- Контрол id=307 «Меню-борд (карточки блюд)» содержит 3 элемента: изображение, название, цена
- Контролы 302-306 имеют `elements: []` (пустые)

## 7. Риски и открытые вопросы

| Риск | Как смягчаем |
|------|-------------|
| Потеря контекста при навигации | Тема сохраняется в localStorage перед переходом. При возврате — загружается актуальное состояние |
| Конфликт изменений (редактор темы и редактор контрола оба открыты) | Навигация через Router разрушает компонент темы — конфликта нет |
| Несохранённые изменения в теме перед переходом | Перед навигацией автоматически сохраняем тему в localStorage (автосохранение) |

## 8. Ожидаемый результат

После реализации:
- В редакторе Темы Электронной очереди, при клике на область с прикреплённым контролом, в правой панели (инспектор области) появляется кнопка «Редактировать контрол»
- По нажатию открывается существующий редактор контрола (`arrivals-control-editor`) с элементами выбранного контрола
- В редакторе контрола можно изменить элементы, их текст, шрифты, размеры, границы
- После сохранения и возврата («Назад») изменения контрола отображаются на canvas темы
- Данные сохраняются в localStorage (персистентность между сессиями)
