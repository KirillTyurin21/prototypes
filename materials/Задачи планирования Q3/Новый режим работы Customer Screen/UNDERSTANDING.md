# Понимание задачи: Прототип «Третий режим CS — Касса не работает»

Дата: 2026-06-10

---

## 1. Что нужно сделать (кратко)

Воспроизвести редактор темы Customer Screen (`http://localhost:3000/prototype/web-screens/themes-cs`) по образцу реальной страницы `http://stand2-iikoweb/customer-screen/index.html#/customer-screen/edit/369`, с доработкой по спецификации: добавить **пятый режим «Касса не работает»** с выбором рекламной кампании.

---

## 2. Анализ спецификации

### 2.1. Суть задачи

Сейчас у Customer Screen 4 режима: Заказ, Итого, Завершение, Ожидание. Когда кассовая смена не открыта, экран в неопределённом состоянии — гость не понимает, работает касса или нет.

**Решение:** добавить 5-й режим «Касса не работает», который автоматически активируется при отсутствии открытой смены и показывает рекламную кампанию (например, «Касса закрыта. Пройдите к кассе №2»).

### 2.2. Что меняется в UI (конструктор тем)

| Аспект | Было | Стало |
|--------|------|-------|
| Выпадающий список режимов | 4 пункта: Экран заказа, Режим ожидания, Экран оплаты, Экран завершения | **5 пунктов** (+ «Касса не работает») |
| Селектор кампании | Отсутствует | Появляется при выборе «Касса не работает» |
| Кампания | Не применимо | Выбор из списка существующих кампаний (необязательное поле) |

### 2.3. Что НЕ делаем (вне scope прототипа)

- Логику плагина (C# — отслеживание смены, переключение режимов)
- Бэкенд-проверки
- Реальные API-запросы

## 5. План реализации

### 5.1. Создать экран `themes-cs-screen.component.ts`

По адресу `http://localhost:3000/prototype/web-screens/themes-cs` (сейчас этого экрана нет).

**Берём за основу** `arrivals-theme-editor-screen.component.ts`, адаптируем:

| # | Доработка | Детали |
|---|-----------|--------|
| 1 | **Список тем CS** | Главный экран: таблица тем (название, разрешение, дата изменения). Кнопка «Добавить» → переход в редактор. Клик по строке → переход в редактор |
| 2 | **Редактор темы** | Копируем движок из arrivals-theme-editor, адаптируем под CS |
| 3 | **Режим «Касса не работает»** | Добавляем в `screenModeOptions`: `{ value: 'closed', label: 'Касса не работает' }` |
| 4 | **Селектор кампании** | При `screenMode === 'closed'` показываем выпадающий список кампаний (мок-данные) |
| 5 | **Preview для закрытого режима** | При mode='closed' canvas показывает заглушку кампании вместо заказа |
| 6 | **Мок-данные** | Список тем CS, кампании, элементы темы |

### 5.2. Регистрация маршрута
