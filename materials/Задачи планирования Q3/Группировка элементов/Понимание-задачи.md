## 1. Суть задачи

### Проблема (из стенограммы, Руслан)

Сейчас палитра элементов в конструкторе тем — это **плоский неструктурированный список** («простыня», «NDP»). 27+ элементов идут подряд без группировки.

### Целевое решение

Реорганизовать палитру элементов в **аккордеон**:
- Элементы группируются по **семантическим категориям**
- Каждая категория — сворачиваемый/разворачиваемый блок
- У каждой категории и каждого элемента — **иконка** (из Material Icons / Lucide)

### UI-паттерн

```
┌─────────────────────────────────┐
│ ДОБАВИТЬ ЭЛЕМЕНТ            [×] │
├─────────────────────────────────┤
│ ▼ Базовые                   2   │  ← категория (аккордеон, раскрыт)
│   🖹 Текст                      │  ← элемент с иконкой
│   🖼 Изображение                │
│ ▶ Контейнеры                2   │  ← категория (свёрнута)
│ ▶ Декоративные              1   │
│ ▶ Медиа                     1   │
│ ▶ Информационные            2   │
└─────────────────────────────────┘
```

## 2. Что меняется в прототипе

### Текущее состояние (файл `arrivals-theme-editor-screen.component.ts`)

В режиме `panelView === 'add-element'` отображается:
- Элемент «Область» (особый, с синим фоном)
- Разделитель «Элементы»
- Плоский список из 27 `elementTypes`

```typescript
// Сейчас: плоский массив
elementTypes: { type: ArrivalsElementType; label: string }[] = [
  { type: 'text', label: 'Текст' },
  { type: 'image', label: 'Изображение' },
  { type: 'order-number', label: 'Номер заказа' },
  // ... ещё 24 элемента
];
```

### Целевое состояние (для прототипа)

Заменяем плоский список на **аккордеон с категориями** для элементов темы Arrivals.

Категории (из спецификации, раздел 5.2):

| Категория | Иконка (Lucide) | Элементы |
|-----------|:---:|----------|
| **Базовые** | `type` | Текст, Изображение |
| **Контейнеры** | `layout-grid` | Область, Всплывающее окно |
| **Декоративные** | `square` | Прямоугольник |
| **Медиа** | `megaphone` | Рекламный блок |
| **Информационные** | `info` | Текущее время, Счетчик |

> **Примечание:** для прототипа группируем только элементы ТЕМЫ (8 шт). Элементы КОНТРОЛА (27 шт) остаются как есть — плоским списком под аккордеоном темовых элементов. Полная группировка контролов — за рамками этого упрощённого прототипа.

## 3. План реализации (7 шагов)

### Шаг 1. Создать тип данных для категорий
**Файл:** `src/app/prototypes/web-screens/types.ts`
- Добавить интерфейс `ElementCategory`:
  ```typescript
  export interface ElementCategory {
    id: string;
    label: string;
    icon: string;
    elements: { type: ArrivalsElementType; label: string; icon: string }[];
    collapsed: boolean;
  }
  ```

### Шаг 2. Создать данные категорий
**Файл:** новый `src/app/prototypes/web-screens/data/element-categories.data.ts`
- Экспортировать массив `ARRIVALS_THEME_CATEGORIES` с 5 категориями и 8 элементами
- Каждой категории и каждому элементу назначить Lucide-иконку

### Шаг 3. Переработать `panelView === 'add-element'` в шаблоне
**Файл:** `arrivals-theme-editor-screen.component.ts`
- Вместо текущего `<div class="element-type-list">` с `*ngFor` по `elementTypes`:
  - Аккордеон категорий (цикл `*ngFor` по `categories`)
  - Каждая категория: кликабельный заголовок (иконка + название + количество + шеврон)
  - При раскрытии — список элементов внутри (иконка + название)
  - Под аккордеоном — разделитель «Элементы контрола» и старый плоский список

### Шаг 4. Добавить логику сворачивания/разворачивания
**Файл:** `arrivals-theme-editor-screen.component.ts`
- Метод `toggleCategory(categoryId: string)` — переключает `collapsed`
- По умолчанию первая категория раскрыта, остальные свёрнуты

### Шаг 5. Стилизовать аккордеон
**Файл:** `arrivals-theme-editor-screen.component.ts` (styles)
- Категория: фон при наведении, плавная анимация раскрытия
- Элемент внутри: отступ слева, иконка 16px, hover-эффект
- Шеврон поворачивается при раскрытии (CSS transition)

### Шаг 6. Обновить `changelog.data.ts`
**Файл:** `src/app/prototypes/web-screens/changelog.data.ts`
- Добавить запись о группировке элементов в палитре

### Шаг 7. Проверить и запустить
- `npm run dev` → открыть редактор тем → «Добавить элемент» → проверить аккордеон

## 4. Что НЕ делаем (ограничения прототипа)

- **НЕ** группируем элементы контрола (27 шт) — только элементы темы (8 шт)
- **НЕ** трогаем бэкенд / API — чисто фронтовое изменение
- **НЕ** создаём новые иконки — используем существующие из Lucide
- **НЕ** меняем логику добавления элемента на canvas — только UI палитры
- **НЕ** затрагиваем другие редакторы (CS, MenuBoard) — только Arrivals

## 5. Ожидаемый результат

После реализации, при нажатии «Добавить элемент» в редакторе тем Электронной очереди, пользователь видит:

1. Аккордеон с 5 категориями элементов темы (с иконками)
2. При клике на категорию — она раскрывается, показывая элементы внутри
3. Клик по элементу — добавляет его на canvas (как и сейчас)
4. Ниже аккордеона — разделитель и плоский список элементов контрола (без изменений)
