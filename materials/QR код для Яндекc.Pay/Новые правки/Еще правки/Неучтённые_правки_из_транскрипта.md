## N-01 — Форма организации: только ИНН

Не запрашивать у клиента, потому что ручной ввод допускает определённую вероятность ошибок, опечаток и так далее. Просто ИНН — сходим в нужные справочники и заполним необходимую информацию.»

| Поле | Переменная | Оставить? |
|------|-----------|-----------|

| Название юрлица | `newPartnerName` | ❌ Убрать |

4. **Список организаций:** Показывать только ИНН и авто-резолвленное название. Без возможности «просмотреть детали» (Kirill: *«Просмотреть мы её тоже никак не можем»*).
5. **Мок DaData:** Создать маппинг ИНН → название для 2–3 демо-значений (для убедительности демо).

## N-02 — MCC-коды: slug + name вместо числовых кодов

### Что сказано на встрече

> **Speaker 2, ~27:51:** «МСС код мы вам отдаём по API. Мы сделали такой эндпоинт… Там будет слаг и name… Вы просто отображаете для пользователя в выпадающем списке, прям с нас по API их стягиваете, он что-нибудь выбирает.»

### Текущее состояние

MCC-коды хранятся как числовые (формат `{ mcc: '5812', name: 'Рестораны' }`).
В dropdown отображается: `5812 - Рестораны`.

### Что нужно сделать

1. **Изменить формат мок-данных** `MOCK_MCC_CODES`:
   ```typescript
   // БЫЛО:
   { mcc: '5812', name: 'Рестораны' }
   
   // СТАЛО:
   { slug: 'restaurants', name: 'Рестораны' }
   ```

2. **Обновить dropdown:** Отображать только `name` (без slug). Пользователь видит «Рестораны», а не «5812 - Рестораны».

3. **Обновить `MerchantEntry.mcc`:** Хранить `slug` вместо числового кода.

4. **Обновить шаблон dropdown:**
   ```html
   <!-- БЫЛО -->
   <option [value]="mcc.mcc">{{ mcc.mcc }} - {{ mcc.name }}</option>
   
   <!-- СТАЛО -->
   <option [value]="mcc.slug">{{ mcc.name }}</option>
   ```

5. **Примерные slug/name пары (на основе реального API Яндекс.Пэй):**
   - `{ slug: 'eat_restaurant', name: 'Рестораны' }`
   - `{ slug: 'eat_bar', name: 'Бары и ночные клубы' }`
   - `{ slug: 'eat_fastfood', name: 'Фастфуд' }`
   - `{ slug: 'beauty_salon', name: 'Салоны красоты' }`
   - `{ slug: 'health_spa', name: 'Оздоровительные и спа-услуги' }`
