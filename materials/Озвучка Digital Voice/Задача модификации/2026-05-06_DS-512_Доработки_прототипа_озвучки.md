# DS-512: Доработки прототипа сервиса озвучки

> **Задача:** DS-512 - Digital Voice: Спецификация сервиса звуков
> **Цель:** Внести доработки в существующий прототип для демонстрации команде
> **Источник требований:** Встреча 04.05.2026 (Руслан, Михаил, Кирилл)
> **Решающее слово:** Руслан (руководитель) и Михаил (PM)

---

## Контекст

Существует работающий прототип сервиса генерации речи (TTS). Сервис уже умеет генерировать речь несколькими голосами. На встрече 04.05.2026 команда определила набор доработок для первого рабочего варианта системы. Ниже - полный перечень доработок, которые необходимо реализовать в прототипе.

**Архитектура:** Сервис генерации - это отдельный микросервис на выделенной машине. Управление сервисом осуществляется из iiko Web (бэкенд). Генерацию инициирует бэкенд, НЕ плагин.

---

## Доработка 1: Справочник звуков (Sound Catalog)

### Что сделать
Создать отдельный раздел интерфейса "Справочник звуков" - это НЕ галерея контента, а отдельная сущность в меню. Справочник показывает только звуковые файлы, сгенерированные для данного клиента (chain-аккаунта).

### Почему
Михаил предложил не смешивать сгенерированные звуки с обычной галереей контента (изображениями, видео). Руслан принял предложение: "Предложение Михаила мы принимаем".

### Требования к реализации
- Справочник - отдельный пункт меню (не внутри галереи)
- Показывает только звуковые файлы (audio)
- Файлы физически хранятся на файловом сервере в системных папках
- Справочник привязан к аккаунту клиента (chain-уровень)

---

## Доработка 2: Структура папок по голосам

### Что сделать
Внутри справочника звуков организовать папочную структуру, разбитую по типам голосов (например: Ксения, Ирина, Василий). Внутри каждого голоса - подпапки для категорий.

### Почему
Руслан определил: "Справочник может быть разбит на наши системные папки, которые называются Ксения, Ирина, Василий - по типу голосов".

### Требования к реализации
Структура папок:
```
[Голос] (например "Ксения")
├── Числа/          - сгенерированные числительные (0-9999)
└── Фразы/          - пользовательские фразы, сгенерированные клиентом
```
- Для каждого доступного голоса создается свой набор папок
- Клиент видит в справочнике только свои сгенерированные звуки
- Внутри каждой папки отображаются звуковые файлы с возможностью прослушивания

---

## Доработка 3: Автогенерация числительных при инициализации сервиса

### Что сделать
При первом запуске сервиса (инициализации для клиента) автоматически сгенерировать все числительные от 0 до 9999 для каждого доступного голоса. Результаты сохраняются в системные папки на файловом сервере.

### Почему
Основной кейс использования - озвучка "Заказ номер N готов". Числа должны быть готовы заранее, чтобы не создавать очередь генерации при каждом заказе. Руслан подтвердил: "Четырехзначные. Как только сервис запускается, он на это все смотрит, нету - значит генерит все". Диапазон до 4 знаков (0-9999).

### Требования к реализации
- При инициализации сервиса проверять наличие числительных для каждого голоса
- Если числительных нет - запускать генерацию всех чисел 0-9999
- Результат сохранять в папку `[Голос]/Числа/` на файловом сервере
- Рекомендация: если часть уже сгенерирована - догенерировать только отсутствующие (на встрече не обсуждалось явно, но логически необходимо)

## Доработка 4: Дефолтные звуки iiko

### Что сделать
Для каждого события должен существовать дефолтный звуковой сигнал от компании iiko. Эти звуки НЕ генерируются TTS - они заранее подготовлены профессионально и захардкожены в системе.

### Почему
Руслан: "Я бы по дефолту просто один какой-то звуковой сигнал сделал, он просто дефолтный, захардкоженный". Кирилл добавил: "Он будет по умолчанию от компании iiko, не сгенерирован, а красиво сделанный". Руслан подтвердил, что они уже существуют: "У нас они уже сгенерированы, у нас уже все есть".

### Требования к реализации
- Для каждого системного события - свой дефолтный звук
- Дефолтные звуки предустановлены и не зависят от TTS-генерации
- Используются как fallback, когда сгенерированная фраза еще не готова (см. Доработку 7)

## Доработка 5: Ввод фразы и отправка на генерацию

### Что сделать
В карточке события пользователь может переключить тип озвучки на "Генерация голоса", написать текст фразы и нажать "Сохранить". После сохранения система ставит фразу в очередь генерации с задержкой ~1 минута.

### Почему
Руслан описал пользовательский сценарий: "Человек заходит в карточку, переводит какое-то событие в голосовой режим, пишет туда фразу". Задержка в ~1 минуту нужна потому, что: "Я могу передумать и могу просто ее исправить по-быстрому" (Руслан). Михаил добавил: "Человек может пять раз подумать и сохранить пять раз".

### Требования к реализации
- В карточке события добавить переключатель типа озвучки: стандартный звук / генерация голоса
- Текстовое поле для ввода фразы (появляется при выборе "генерация голоса")
- Возможность указать место для номера заказа в фразе (например: "Заказ номер {номер} готов")
- При нажатии "Сохранить" фраза попадает в очередь генерации
- Задержка перед отправкой на сервер генерации: ~1 минута (чтобы пользователь мог отменить или отредактировать)
- В течение задержки фраза может быть отменена или изменена

## Доработка 6: Панель очереди генерации (Generation Queue Panel)

### Что сделать
Создать выпадающую панель в верхней части интерфейса, которая показывает текущую очередь генерации. Панель раскрывается по клику на индикатор. Когда генерация активна - показывается анимация прогресса.

### Почему
Кирилл предложил концепцию: "Выпадающая панель очереди сверху. Там будет 2-3-4 элемента. Максимально интуитивно". Руслан согласился: "Похож на план". Руслан также настаивал: "Должна быть кнопка статус, на которой видно процесс генерации. Должно быть четкое понимание - что-то происходит или нет".

### Требования к реализации

**Индикатор в верхней панели:**
- Когда ничего не генерируется - индикатор не отображается (или неактивен)
- Когда идет генерация - горизонтальная анимированная полоса прогресса (Кирилл: "горизонтальная полоска, как новогодняя трубочка, красная движется в правую сторону")
- Клик по индикатору раскрывает выпадающую панель

**Содержимое выпадающей панели:**
- Список всех фраз в очереди генерации
- Для каждого элемента показывать:
  - Текст фразы
  - Статус: ожидает отправки / генерируется / готово / ошибка
  - Если еще не отправлена (в задержке ~1 мин) - кнопка отмены (иконка корзины)
  - Если готова - кнопка воспроизведения (прослушать результат прямо в панели)

**Управление из панели:**
- Отмена фразы до отправки на генерацию (корзина) - фраза удаляется из очереди, нагрузка на генератор не создается
- Прослушивание готовой фразы (play-кнопка) - прямо в панели, без перехода в другой раздел
- Переход к редактированию фразы из панели (если нужно поменять и перегенерировать)
- Повторная отправка при ошибке генерации

## Доработка 7: Механизм fallback (приоритет источника звука)

### Что сделать
Реализовать автоматическое переключение между источниками звука на терминале. Если для события есть сгенерированная фраза - использовать ее. Если фразы еще нет (не сгенерирована) - использовать стандартный звук (делень/дефолт). Переключение должно быть автоматическим, без ручного вмешательства пользователя.

### Почему
Руслан принял решение (отклонив предложение Михаила о блокировке): "Если есть фраза - говорим фразу. Нет фразы - используем старый вариант". И далее: "Когда у меня на терминал это долетает, терминал такой - о, у меня генерация, но файлов пока нет, я использую старый делень. Как только файл прилетает - я начинаю использовать". Централизованная активация отклонена в пользу автоматического переключения.

### Требования к реализации
- Логика выбора источника звука:
  - Если для события есть сгенерированная фраза - использовать ее
  - Если сгенерированной фразы нет - использовать то, что было настроено ранее (старый звук / делень)
  - Если ничего не было настроено (пользователь начинает с нуля) - использовать дефолтный звук iiko
- Переключение автоматическое: как только сгенерированный файл доставлен на терминал, он сразу начинает использоваться
- Пока файл не доставлен - продолжает работать предыдущий источник
- НЕ требуется ручная активация или "кнопка запуска"
- Пользователь может менять тип озвучки в карточке события даже если событие уже назначено на терминал

## Доработка 8: Быстрая прослушка голоса (Voice Preview)

### Что сделать
Добавить возможность быстрого прослушивания образца голоса - заранее подготовленный демо-трек для каждого типа голоса. Это НЕ конкретная фраза пользователя, а стандартный пример того, как звучит данный голос (включая числа, фразы и интонации).

### Почему
Михаил предложил: "Мы можем стандартную фразу сгенерить и показать - вот так приблизительно будет выглядеть ваш голос. Человек сразу поймет, подходит ему этот голос, подходит ли генерация с нотками робота". Руслан уточнил: "Быстрая прослушка по типу голоса, просто фразы. Вообще, не именно фраза, которую он написал, а рандомно. Вместе с числами, цифрами". Кирилл подтвердил: "Заранее загруженная дорожка".

### Требования к реализации
- Для каждого доступного голоса - заранее подготовленная демо-дорожка
- Демо включает пример фразы + числа (чтобы показать качество произношения)
- Кнопка "Прослушать" рядом с выбором голоса
- Воспроизведение мгновенное (файл уже загружен, без обращения к генератору)
- Цель: быстрая оценка голоса до принятия решения о генерации

## Доработка 9: Редактирование и перегенерация из окна статуса

### Что сделать
В панели очереди генерации (или в справочнике звуков) пользователь должен иметь возможность: прослушать готовую фразу, изменить текст фразы, отправить на повторную генерацию. Все - без перехода в карточку события.

### Почему
Руслан: "Завтра утром зашел, вижу все свои сгенеренные истории и могу прослушать. Если что-то не устроило - могу тут же это поменять, прям тут же в этом окне статуса. Чтобы не лезть куда-то, не искать. Сразу же тут же поменял фразу и опять на перегенерацию запустил. А система работает по-старому".

### Требования к реализации
- В панели статуса (или справочнике) для каждой сгенерированной фразы:
  - Кнопка "Прослушать" - воспроизвести текущий вариант
  - Кнопка "Редактировать" - открыть поле редактирования текста фразы
  - Кнопка "Перегенерировать" - отправить измененную фразу заново на генерацию
- Во время перегенерации система продолжает использовать старый (текущий) вариант звука
- Новый вариант заменит старый автоматически после завершения генерации

## Что НЕ входит в текущую итерацию

- Выбор конкретного размещения настроек в интерфейсе iiko Web (Руслан: "Не предлагать пока, где они должны размещаться, как они должны выглядеть")
- Уведомления на почту о готовности генерации (обсуждалось, но не утверждено)
- Централизованная кнопка "Активировать озвучку" (отклонено Русланом в пользу автоматического fallback)
- Блокировка назначения события на терминал до завершения генерации (отклонено Русланом: "Зачем назначать заново? Он либо есть, либо его нет")
- Интеграция с Яндекс SpeechKit (отложена, задача PB-3933)
- Контроль объема хранилища (Руслан упомянул, но отложил: "Детализация это множество")
