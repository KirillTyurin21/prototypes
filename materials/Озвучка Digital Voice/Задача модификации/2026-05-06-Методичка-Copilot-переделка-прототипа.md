## Контекст

Прототип интерфейса Озвучки в iikoWeb был создан по решениям встречи #1 (4 мая 2026, Кирилл + Михаил + Руслан). Прототип включал: справочник звуков, автогенерацию 0-9999 числительных, панель очереди с управлением, fallback-механизм, preview голоса.

На встрече #2 (6 мая 2026) с участием разработчиков была утверждена новая архитектура (Схема Максима), которая существенно меняет подход к генерации речи. Часть решений встречи #1 отменена, часть скорректирована, появились новые решения.

Эта методичка описывает ВСЕ изменения прототипа, которые необходимо внести по итогам встречи #2.

## Детальные инструкции

### Изменение 1. УБРАТЬ: Справочник звуков (Sound Catalog)

**Что было в прототипе (встреча #1):**
Отдельный раздел интерфейса "Справочник звуков" - показывает все звуковые файлы, сгенерированные для данного клиента. Привязан к chain-аккаунту. Файлы хранятся на файловом сервере.

**Решение встречи #2:**
Руслан: *"Справочник вот этих вот звуков мы убираем."*

**Действие:**
- Удалить раздел/страницу "Справочник звуков" из интерфейса
- Удалить все навигационные ссылки на справочник (меню, хлебные крошки)
- Информация о генерации теперь показывается в двух местах: таблица обработчиков (основная) и лог-окно очереди (статусы)

**Обоснование:** в новой архитектуре (Схема Максима) куски хранятся в приватном кэше микросервиса, недоступном пользователю. Готовые фразы хранятся в Галерее. Отдельный справочник не нужен.

**Зависимости:** Изменение 9 (кнопка "Проиграть") и Изменение 10 (размер) заменяют функции, которые были в справочнике.

---

### Изменение 2. УБРАТЬ: Структура папок по голосам

**Что было в прототипе (встреча #1):**
Иерархия папок: `[Голос] / Числа/` и `[Голос] / Фразы/` для каждого голоса (Ксения, Василий и др.). Клиент видел свои сгенерированные файлы в этой структуре.

**Решение встречи #2:**
Куски хранятся в приватном кэше микросервиса. Пользователь не видит внутреннюю структуру кэша.

**Действие:**
- Удалить UI-элементы, связанные с иерархией папок по голосам
- Удалить навигацию по папкам Числа/Фразы
- Пользователь видит только таблицу обработчиков со статусами и готовыми фразами

**Зависимости:** связано с Изменением 1 (справочник, где отображались папки).

---

### Изменение 3. ЗАМЕНИТЬ: Автогенерация числительных

**Что было в прототипе (встреча #1):**
Автогенерация чисел 0-9999 для каждого голоса при инициализации сервиса. Результат сохраняется в папку `[Голос]/Числа/` на файловом сервере.

**Решение встречи #2:**
Генерируется только 27 базовых файлов на голос: единицы (1-9), десятки (10-90), сотни (100-900). Любое число собирается из этих базовых файлов на лету. Например, число 321 = файл "300" + файл "20" + файл "1".

**Действие:**
- Заменить логику "генерация 10 000 файлов" на "генерация 27 базовых файлов на голос"
- Если в прототипе есть прогресс-бар инициализации с отображением "Генерация 0-9999" - заменить на "Генерация базовых числительных (27 файлов)"
- Если в прототипе есть UI для просмотра сгенерированных чисел - убрать (это часть справочника, Изменение 1)

**Контекст для Copilot:** сборка числа из базовых файлов происходит на плагине iikoFront, а не в iikoWeb. В интерфейсе Admin-панели эта логика не отображается. Но если есть индикатор прогресса генерации - он должен учитывать 27 файлов, а не 10 000.

---

### Изменение 4. ИЗМЕНИТЬ: Формат переменных в шаблоне фразы

**Что было в прототипе (встреча #1):**
Фигурные скобки: `Заказ номер {номер} готов`

**Решение встречи #2:**
Руслан: *"Потому что у клиента круглые скобки часто люди используют, квадратные совсем не используют."*

Квадратные скобки: `Заказ [order_number] готов, подойдите к выдаче`

**Действие:**
- Заменить формат переменных в шаблонах фраз: `{...}` ->> `[...]`
- Обновить placeholder текстового поля ввода фразы: показать пример с квадратными скобками
- Обновить примеры/подсказки в интерфейсе, если есть
- Обновить валидацию: парсер переменных должен искать `[имя_переменной]`, а не `{имя_переменной}`
- Использовать английские имена переменных в скобках: `[order_number]`, `[name]`, а не кириллические

**Зависимости:** связано с Изменением 5 (кнопка автоподстановки вставляет переменную в этом формате).

---

### Изменение 5. ДОБАВИТЬ: Кнопка автоподстановки переменных

**В прототипе (встреча #1):** не было.

**Решение встречи #2:**
Рядом с полем ввода шаблона фразы добавить кнопки/список переменных. Клик по кнопке вставляет переменную в текущую позицию курсора.

**Действие:**
- Добавить под/над/рядом с полем ввода фразы набор кнопок с доступными переменными
- Список переменных: `[order_number]`, `[name]` (минимальный набор; полный список определяется в спецификации)
- При клике по кнопке - текст переменной вставляется в позицию курсора в текстовом поле
- Если курсор не установлен - вставка в конец текста

**Визуальное оформление:**
- Кнопки компактные (chip/tag-стиль), расположены горизонтально
- Каждая кнопка содержит имя переменной: `order_number`, `name`
- При наведении - подсказка о назначении переменной

**Зависимости:** Изменение 4 (формат `[...]`) определяет, в каком формате вставляется переменная.

---

### Изменение 6. ИЗМЕНИТЬ: Панель очереди генерации ->> read-only

**Что было в прототипе (встреча #1):**
Панель очереди с полным управлением:
- Кнопка отмены (иконка корзины) для фраз в задержке
- Кнопка воспроизведения готовых фраз
- Переход к редактированию фразы
- Повторная отправка при ошибке
- Анимированная полоска прогресса в верхней панели

**Решение встречи #2:**
Руслан: *"Управление в карточке оставляем. То есть, если я не хочу, чтобы у меня ничего генерировалось, я должен зайти в карточку и выбрать, что это у меня не генерация, а файл."*

**Действие:**
- Убрать из панели очереди ВСЕ кнопки управления: отмена, воспроизведение, редактирование, повторная отправка
- Панель очереди становится лог-окном только для просмотра статусов - read-only
- Добавить клик по элементу очереди ->> открытие карточки обработчика (см. Изменение 8)
- Анимированную полоску прогресса в верхней панели - ОСТАВИТЬ (индикатор наличия активных генераций)
- Для каждого элемента очереди отображать: имя обработчика, текст фразы, статус генерации

**Зависимости:** Изменение 8 (клик открывает карточку), Изменение 11 (управление в карточке).

---

### Изменение 7. ИЗМЕНИТЬ: Статусы очереди генерации

**Что было в прототипе (встреча #1):**
4 статуса: Ожидает отправки | Генерируется | Готово | Ошибка

**Решение встречи #2:**
Руслан: *"Я просто хотел бы как человек видеть и понимать: он еще в очереди, он начал, ну то есть процесс генерации начался, ну то есть генерируется, да, и готово, все - вот три статуса я хочу видеть."*

3 статуса: **Ожидает** | **Генерируется** | **Готово**

**Действие:**
- Заменить 4 статуса на 3: убрать статус "Ошибка"
- Статус "Ожидает отправки" переименовать в "Ожидает"
- Визуальные индикаторы (цвет/иконка) - три состояния:
  - Ожидает - нейтральный (серый/синий)
  - Генерируется - активный (оранжевый/анимация)
  - Готово - завершенный (зеленый/галочка)
- Если в прототипе есть обработка ошибок с визуальным отображением статуса "Ошибка" - убрать

> Примечание: это не значит, что ошибки генерации невозможны. Обработка ошибок будет описана в спецификации отдельно. На уровне прототипа UI достаточно трех статусов.

### Изменение 8. ДОБАВИТЬ: Клик по элементу очереди открывает карточку

**В прототипе (встреча #1):** элементы очереди имели собственные кнопки управления.

**Решение встречи #2:** вместо кнопок - клик по элементу очереди открывает карточку обработчика, где уже есть все инструменты управления.

**Действие:**
- Каждый элемент в панели очереди генерации сделать кликабельным
- При клике - открывать карточку обработчика, к которому относится данная генерация
- Визуальная индикация кликабельности: курсор-pointer, подсветка при наведении

**Зависимости:** Изменение 6 (панель read-only), Изменение 11 (управление в карточке).

### Изменение 9. ДОБАВИТЬ: Кнопка "Проиграть" в таблице обработчиков

**В прототипе (встреча #1):** кнопка воспроизведения была в панели очереди и в справочнике звуков.

**Решение встречи #2:**
Руслан: *"Переделываешь визуализацию, добавляешь в обработчик колоночку проиграть звук. И оттуда мы его проигрываем, этот звук, если он сгенерен."*

**Действие:**
- В таблице обработчиков событий (страница "Список коллекций" ->> раскрытая коллекция ->> вложенные обработчики) добавить столбец с кнопкой "Проиграть"
- Кнопка активна только для обработчиков с типом "Генерация" и статусом "Готово"
- Кнопка неактивна (disabled/скрыта) для обработчиков с типом "Файл" или статусом "Ожидает"/"Генерируется"
- При нажатии - воспроизвести сгенерированную фразу в браузере (с дефолтным номером заказа)
- Визуал: иконка play (треугольник) или кнопка-текст "Проиграть"

**Контекст для Copilot:** при нажатии iikoWeb отправляет запрос микросервису, который собирает фразу с дефолтным номером (300) и возвращает аудиофайл. Фраза воспроизводится в браузере клиента.

**Зависимости:** Изменение 1 (кнопка заменяет функционал из справочника).

---

### Изменение 10. ДОБАВИТЬ: Столбец "Размер" в таблице обработчиков + итого

**В прототипе (встреча #1):** не было.

**Решение встречи #2:**
Руслан: *"И размер надо вынести туда в табличку... Чтобы человек понимал... Я бы даже тогда суммировал бы по столбцу и показывал итого."*

**Действие:**
- В таблице обработчиков добавить столбец "Размер"
- Для обработчиков с типом "Генерация" и статусом "Готово" - показывать размер сгенерированной фразы (в КБ или МБ)
- Для обработчиков с типом "Файл" - показывать размер прикрепленного звукового файла
- Для обработчиков со статусом "Ожидает"/"Генерируется" - прочерк или "..."
- Внизу таблицы (или в подвале столбца) показывать суммарный размер всех фраз: "Итого: XX МБ"
- Возможность удаления сгенерированных фраз для освобождения места (кнопка удаления в карточке обработчика или контекстном меню)

**Зависимости:** ни от чего не зависит, можно реализовать параллельно.

---

### Изменение 11. ИЗМЕНИТЬ: Управление генерацией - только из карточки обработчика

**Что было в прототипе (встреча #1):**
Управление было распределено:
- Панель очереди: отмена, повторная отправка, прослушивание
- Справочник звуков: прослушивание, редактирование, перегенерация

**Решение встречи #2:**
Все управление генерацией сосредоточено в одном месте - в карточке обработчика.

**Действие:**
Карточка обработчика (страница создания/редактирования) должна содержать:

- **Переключатель типа озвучки:** "Файл" / "Генерация"
  - Если "Файл" - поле выбора файла из Галереи (как в v1.0)
  - Если "Генерация" - текстовое поле для ввода шаблона фразы + выбор голоса + кнопки переменных
- **Текстовое поле шаблона фразы** (видно только при типе "Генерация"):
  - Placeholder: `Например: Заказ [order_number] готов`
  - Кнопки автоподстановки переменных (см. Изменение 5)
- **Выбор голоса** (видно только при типе "Генерация"):
  - Dropdown/select с доступными голосами (Ксения, Василий и др.)
  - Кнопка preview голоса рядом (если реализовано, см. НЕ МЕНЯТЬ #14)
- **Статус генерации** (видно только при типе "Генерация"):
  - Текущий статус: Ожидает / Генерируется / Готово
  - Кнопка "Проиграть" (если статус "Готово")
- **Кнопка "Сохранить":** при сохранении с типом "Генерация" - фраза ставится в очередь генерации (с задержкой)

Смена типа с "Генерация" на "Файл" - отменяет текущую генерацию (если она в процессе). Смена с "Файл" на "Генерация" - запускает новую генерацию.

**Зависимости:** Изменение 4 (формат переменных), Изменение 5 (кнопки переменных), Изменение 6 (панель read-only).

---

### Изменение 12. ДОБАВИТЬ: Выбор голоса в карточке обработчика

**В прототипе (встреча #1):** выбор голоса мог быть привязан к справочнику звуков или карточке обработчика (уточнить).

**Решение встречи #2:**
Голос выбирается в карточке обработчика при типе "Генерация".

**Действие:**
- Добавить dropdown/select "Голос" в карточку обработчика
- Поле видно только при типе озвучки "Генерация"
- Список голосов: Ксения, Василий и другие доступные (загружаются с сервера)
- Голос обязателен для заполнения при типе "Генерация"
- Смена голоса ->> перегенерация фразы выбранным голосом (старая фраза заменяется новой)

**Зависимости:** Изменение 11 (карточка обработчика).

---

### НЕ МЕНЯТЬ #13: Fallback-механизм

**Решение встречи #1 остается в силе:**
1. Если для события есть сгенерированная фраза ->> использовать ее
2. Если нет ->> использовать ранее настроенный звук
3. Если ничего не настроено ->> дефолтный звук iiko

Переключение автоматическое. Встреча #2 не отменяла это решение.

**Действие:** не трогать реализацию fallback в прототипе.

---

### НЕ МЕНЯТЬ #14: Voice Preview (демо-трек голоса)

**Решение встречи #1 остается в силе:**
Для каждого голоса - заранее подготовленная демо-дорожка. Кнопка "Прослушать" рядом с выбором голоса. Мгновенное воспроизведение без обращения к генератору.

Встреча #2 не отменяла это решение.

**Действие:** не трогать реализацию preview голоса в прототипе. При необходимости - перенести кнопку preview ближе к dropdown выбора голоса в карточке обработчика (Изменение 12).

## Архитектурный контекст (для понимания изменений)

Copilot, работающий с прототипом, должен понимать новую архитектуру, чтобы правильно отображать UI-элементы.

### Схема Максима (утверждена на встрече #2)

**Принцип:** микросервис разбивает фразы на атомарные куски (слова, словосочетания), кэширует их кросс-аккаунтно. Плагин на терминале получает куски и собирает итоговую фразу на лету.

**Три хранилища:**

| Хранилище | Что хранит | Видимость для пользователя |
|-----------|-----------|---------------------------|
| Кэш кусков (микросервис) | Атомарные куски всех голосов | Невидим. Пользователь не имеет доступа |
| Галерея iikoWeb | Готовые собранные фразы (с дефолтным номером) | Виден. Пользователь может прослушать и удалить |
| Локальный кэш (плагин) | Куски фраз на терминале | Невидим. Управляется плагином автоматически |

**Что это значит для UI прототипа:**
- Пользователь НЕ видит куски фраз, папки по голосам, внутреннюю структуру кэша
- Пользователь ВИДИТ: статус генерации (Ожидает/Генерируется/Готово), кнопку "Проиграть" готовой фразы, размер фразы
- Пользователь УПРАВЛЯЕТ: в карточке обработчика (текст фразы, голос, тип озвучки)

### Предсгенерация цифр

27 файлов на голос (1-9, 10-90, 100-900). Генерируются при инициализации микросервиса. В UI это не отображается - пользователь просто видит, что фразы с номерами работают.

### Быстрый канал для имен

Если фраза содержит переменную `[name]`, плагин на терминале запрашивает озвучку имени напрямую у микросервиса (без очереди). Генерация на лету. В Admin-панели iikoWeb это никак не отображается - работа происходит на уровне плагин-микросервис.

### Очередь микросервиса

Без внешних брокеров (Kafka/RabbitMQ). Своя очередь на БД. N экземпляров микросервиса. Статусы: Принят ->> В обработке ->> Готово. В Admin-панели пользователь видит три статуса (Ожидает/Генерируется/Готово) - это маппинг внутренних статусов микросервиса на пользовательские.

## Зависимости между изменениями

```
Изменение 1 (Убрать справочник)
  └── Изменение 2 (Убрать папки по голосам) - убираются вместе
  └── Изменение 9 (Кнопка "Проиграть") - замещает функцию справочника
  └── Изменение 10 (Размер) - замещает функцию справочника

Изменение 4 (Квадратные скобки)
  └── Изменение 5 (Кнопка автоподстановки) - вставляет в формате [...]

Изменение 6 (Панель очереди read-only)
  └── Изменение 7 (3 статуса) - статусы для read-only панели
  └── Изменение 8 (Клик открывает карточку) - навигация из панели
  └── Изменение 11 (Управление в карточке) - куда перенесено управление

Изменение 11 (Карточка обработчика - центр управления)
  └── Изменение 4 (Формат переменных)
  └── Изменение 5 (Кнопка автоподстановки)
  └── Изменение 12 (Выбор голоса)
```

**Рекомендуемый порядок реализации:**
1. Изменения 1, 2 (убрать справочник и папки)
2. Изменения 4, 5 (формат переменных + автоподстановка)
3. Изменения 6, 7, 8 (панель очереди ->> read-only + 3 статуса + клик)
4. Изменения 11, 12 (карточка обработчика + выбор голоса)
5. Изменения 9, 10 (кнопка "Проиграть" + размер в таблице)

## Итоговая таблица обработчиков (целевое состояние)

После всех изменений таблица обработчиков в раскрытой коллекции должна содержать:

| Столбец | Описание | Для типа "Файл" | Для типа "Генерация" |
|---------|----------|-----------------|---------------------|
| Название | Имя обработчика | Текст | Текст |
| Тип озвучки | Файл или Генерация | "Файл" | "Генерация" |
| Кол-во событий | Сколько событий привязано | Число | Число |
| Статус | Статус генерации | Пустая ячейка (прочерк) | Ожидает / Генерируется / Готово |
| Проиграть | Кнопка воспроизведения | Пустая ячейка (прочерк) | Кнопка play (активна при "Готово") |
| Размер | Размер файла/фразы | Размер загруженного файла | Размер фразы (при "Готово") |
| Действия | Редактировать, Дублировать, Удалить | Кнопки | Кнопки |

Внизу таблицы: **Итого: XX МБ** (сумма по столбцу "Размер").
