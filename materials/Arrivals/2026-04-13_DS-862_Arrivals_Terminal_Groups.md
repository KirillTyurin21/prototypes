## КРАТКОЕ ОПИСАНИЕ ЗАДАЧИ

Плагин Arrivals (электронная очередь) в iikoFront отображает заказы на дисплее для клиентов ресторана. В текущей реализации на дисплее отображаются **все** заказы без возможности фильтрации. Необходимо проработать и специфицировать поддержку **терминальных групп** - добавить в настройки дисплея Arrivals ещё один список, в котором пользователь выбирает терминальные группы, и на дисплее отображаются только заказы, относящиеся к выбранным терминальным группам.

## ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Архитектура плагина Arrivals

Arrivals - это **плагин iikoFront** (C#), работающий совместно с **веб-клиентом** (Angular) и **бэкендом** (iikoWeb). Архитектура:

```
+------------------+     +----------------+     +-------------------+
|   iikoWeb        |     |  Plugin        |     | Web-клиент        |
|   (Backend)      |<--->|  Arrivals C#   |<--->| (Angular 16)      |
|                  |     |  (iikoFront)   |     | (Дисплей Arrivals)|
+------------------+     +----------------+     +-------------------+
        |                        |
        v                        v
+------------------+     +-------------------+
| База данных      |     | iikoFront API V9  |
| (настройки,      |     | (IOperationService)|
| темы, экраны)    |     |                   |
+------------------+     +-------------------+
```

**Основные компоненты:**

1. **Backend (iikoWeb):**
   - Хранит настройки экранов/дисплеев Arrivals
   - API для управления терминалами, экранами, темами
   - Endpoint `/api/arrivals/theme-ouc` - получение настроек темы
   - Механизм ревизий для отслеживания изменений (DS-749)
   - Обработка активации/деактивации экранов (DS-791)

2. **Plugin (C# плагин iikoFront):**
   - Получает настройки от iikoWeb
   - Взаимодействует с iikoFront API V9
   - Передаёт данные заказов веб-клиенту
   - Обрабатывает сообщения от iikoWeb (DS-776)
   - Текущая версия: Arrivals.V8.1.0.97 (из DS-807)

3. **Frontend (Angular 16):**
   - Отображение заказов на дисплее
   - Страница "Настройка экранов" (DS-772, DS-775, DS-824)
   - Страница "Настройка терминалов" (DS-824)
   - Конструктор тем с контролами
   - Предпросмотр тем (DS-813)
   - Поддержка кастомных шрифтов (DS-830)

### Терминальные группы в iiko

**Терминальная группа (Terminal Group)** - это логическая группировка терминалов iiko в одном ресторане. Каждый ресторан может иметь несколько терминальных групп.

**Ключевые сведения из iikoFront SDK (API V9):**

| Метод | Описание |
|-------|----------|
| `GetTerminalsGroups()` | Получить список ВСЕХ терминальных групп |
| `GetTerminalsGroupById(id)` | Получить терминальную группу по ID |
| `GetHostTerminalsGroup()` | Терминальная группа текущего терминала |
| `GetTerminalsGroupTerminals(group)` | Список терминалов в группе |
| `GetTerminalsGroupRestaurantSections(group)` | Секции ресторана для терминальной группы |
| `GetTerminalsGroupPointsOfSale(group)` | Точки продаж терминальной группы |
| `TryGetTerminalsGroupById(id)` | Безопасное получение группы (null если нет) |
| `GetTerminals()` | Полный список терминалов |
| `GetTerminalById(id)` | Терминал по ID |
| `GetHostTerminal()` | Текущий терминал |

**Важно:** В iikoTransport терминальные группы синхронизируются через очереди:
- `transport.queue.distributor.dictionaries` - уведомление об изменении данных в терминальных группах (Terminal Groups Updated)
- Terminal Groups Reload - полный список терминальных групп (актуализация локального хранилища TerminalGroups)

### Модель данных

**Терминальная группа (ITerminalsGroup в iikoFront SDK):**
- `Id` (Guid) - уникальный идентификатор
- `Name` (string) - название группы

**Ключевые связи:**
- Терминальная группа содержит N терминалов
- Каждый терминал принадлежит одной терминальной группе
- Заказы создаются на конкретном терминале, который входит в терминальную группу

### Текущие настройки дисплея Arrivals

Из анализа существующих задач (DS-824, DS-775):
- **Страница "Настройка терминалов"**: содержит список терминалов Arrivals с чекбоксами
- **Страница "Настройка экранов"**: содержит кнопки "Обновить настройки плагина" и "Сделать скриншот"
- Каждый дисплей Arrivals привязан к терминалу
- Есть механизм ревизий (revision) для отслеживания изменений настроек (DS-749)

### Конструктор тем (контролы)

Arrivals использует конструктор тем с контролами:
- **Контрол электронной очереди** - основной контрол для отображения заказов
- **Контрол externalData** (DS-826) - новый элемент для отображения произвольных данных из ExternalData заказа
- **Всплывающее окно** - popup с информацией о заказе (DS-807, DS-813)
- **Системные настройки** - общие параметры отображения

Контролы задаются через JSON-конфигурацию (data.json).

## ВАЖНЫЕ ЗАМЕЧАНИЯ

### Что НЕ делать:
1. **НЕ путать терминальные группы с группами продуктов** - это разные сущности
2. **НЕ менять существующую логику отображения заказов** - фильтрация по терминальным группам ДОБАВЛЯЕТСЯ к существующему функционалу
3. **НЕ ломать обратную совместимость** - если терминальные группы не выбраны, дисплей должен работать как раньше (показывать все заказы)

### Граничные случаи:
1. **Пустой выбор** (ни одна группа не выбрана) - показывать все заказы или ничего? (ТРЕБУЕТСЯ РЕШЕНИЕ)
2. **Единственная терминальная группа** - имеет ли смысл показывать выбор?
3. **Изменение конфигурации терминальных групп** - что происходит если группа удалена, а ссылка осталась?
4. **Ревизии** - обновление ревизии терминала при изменении привязки к терминальным группам (по аналогии с DS-749)
5. **Лицензии** - нужна ли лицензия на эту функциональность? (по аналогии c DS-801: лицензия 21105209 для балансира)

### Контекст разговора 10.04.2026 (Руслан):

- Руслан хочет аналогичное свойство для **кухонного заказа**
- Это может быть связано с фильтрацией Arrivals - нужно понимать принадлежность заказа к терминальной группе
